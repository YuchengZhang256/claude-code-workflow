# Pipeline bugs are tracked in Linear project INGEST

**What**: All data-ingest / ETL pipeline bugs live in the Linear project
`INGEST`. Ticket IDs look like `ING-<number>`.

**Why**: This is the authoritative issue tracker for pipeline work.
Asking the user for "bug context" without first checking this project
wastes their time.

**How to apply**: When the user references a pipeline bug, a numeric ID,
or a stalled ingest job without giving full context, ask whether they
have a Linear ticket (`ING-<number>`) before starting debugging from
scratch. Do not attempt to open Linear directly from this tool — just
cite the ticket in follow-up writing so the user can cross-reference it.
